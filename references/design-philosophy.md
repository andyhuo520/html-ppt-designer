# 设计哲学与智能布局决策系统 v1.0

> 源自 MindCanvas 设计体系精华，与 HTML PPT Designer v3.0 融合使用。
> 本文档是对现有 layout-patterns.md 和 color-system.md 的**补充**，不替代它们。
> 重点：内容驱动的动态布局决策、密度控制、设计思维检查清单。

---

## 一、核心设计宣言

### 1. 动态内容驱动，而非模板套用

不应简单套用固定模板，而应根据内容的语义和结构，选择最能有效传达信息的布局方式。每一页 Slide 都应是一个独立且精致的设计作品。

### 2. Apple 级别设计哲学

- **极致简约**：移除冗余，聚焦核心
- **无比精致**：每个像素都有意义
- **注重细节**：精确控制字号、行高、字间距、元素间距
- **形式与功能统一**：设计服务于信息传达，视觉增强理解

### 3. 反 AI 审美铁律（保留自 v3.0）

- 禁止千篇一律的圆角卡片堆叠
- 禁止毫无根据的 glow/blur 装饰
- 禁止图标当视觉主角
- 每一处留白都是有意图的"静默表达"
- 版式的张力来自"不对称的平衡"

## 二、内容密度控制规则（最高优先级）

### 核心原则：一页一个核心观点

| 规则 | 阈值 | 处理方式 |
|------|------|---------|
| 段落过多 | > 3 个主要段落 | 必须拆页 |
| 列表过长 | > 7-8 个列表项 | 必须拆页（每页 3-4 项） |
| 阅读时长 | > 30 秒 | 必须拆页 |
| 页面占用 | > 80% 页面高度 | 必须拆页 |
| 信息点数 | > 6 个信息点/页 | 必须拆页（标准模式）|

### 拆页策略

```
宁可多页，不可过载
─────────────────────
优先保证每页的低信息密度和视觉美观，
绝不为节省页数而牺牲清晰度。
```

### 页面类型与信息密度对照

| 页面类型 | 最大信息点 | 文字量上限 | 留白比例 |
|---------|-----------|----------|---------|
| 封面页 | 1（标题+副标题） | 20 字以内 | ≥ 60% |
| 金句页 | 1 句核心观点 | 30 字以内 | ≥ 70% |
| 内容页（简） | 3 个要点 | 100 字以内 | ≥ 40% |
| 内容页（详） | 5-6 个要点 | 200 字以内 | ≥ 35% |
| 数据页 | 3-4 个 KPI | 数字+标签 | ≥ 40% |
| 对比页 | 2 组对比项 | 各 3-4 点 | ≥ 30% |
| 结尾页 | CTA + 联系方式 | 15 字以内 | ≥ 60% |

## 三、智能布局策略选择决策树

### 内容类型 → 布局策略映射

```
用户内容输入
    ↓
┌─ 内容类型识别 ─────────────────────────────────┐
│                                                 │
│  单一核心观点/金句？                              │
│     → 单点聚焦布局（大字号、多留白）               │
│                                                 │
│  标题 + 段落描述？                                │
│     → 全屏背景图 + 居中文字（TED 风格）            │
│     → 或左字右图（Keynote 风格）                   │
│                                                 │
│  多个并列特点/要素？                              │
│     → 网格/模块化布局（2×2 或 3 列）              │
│     → 或中心焦点 + 侧翼支撑                       │
│                                                 │
│  步骤/流程/历程？                                 │
│     → 时间轴布局（横向优先）                       │
│     → 或纵向流程图                                │
│                                                 │
│  两组对比信息？                                   │
│     → 对比/并列布局（Before-After）               │
│                                                 │
│  层级/重要性关系？                                │
│     → 分层/金字塔布局                             │
│                                                 │
│  数据/数值展示？                                  │
│     → 数据卡片 + SVG 图表                        │
│                                                 │
│  概念/结构解释？                                  │
│     → 图文结合（SVG 图形 + 文字说明）              │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 布局策略详细定义

#### 1. 单点聚焦布局
- **适用**：金句、核心结论、重要数字
- **特征**：字号 ≥ 48px（正文区域）、留白 ≥ 70%
- **避免**：添加任何"装饰性"辅助内容
- **CSS 要点**：`display: flex; align-items: center; justify-content: center;`

#### 2. 中心焦点 + 侧翼支撑
- **适用**：一个核心概念 + 2-4 个支撑要素
- **特征**：中心元素占 40-50% 视觉重量，侧翼平衡分布
- **CSS 要点**：中心 `grid-column: span 2` 或绝对定位居中

#### 3. 时间轴/流程图（横向优先）
- **适用**：发展历程、操作步骤、项目阶段
- **特征**：节点 + 连接线 + 简短标签，水平方向阅读
- **注意**：步骤 ≤ 5 个/页，超出则拆页

#### 4. 分层/金字塔
- **适用**：优先级、层级关系、战略框架
- **特征**：从顶部到底部逐层展开，最重要在顶部
- **SVG 实现**：用 SVG 绘制三角形或梯形分层

#### 5. 对比/并列
- **适用**：方案比较、优劣分析、Before/After
- **特征**：严格的左右或上下对称布局
- **色彩**：用不同色块区分两侧（如灰 vs 主色）

#### 6. 网格/模块化
- **适用**：多个同等重要的信息块、特征列表
- **特征**：2×2 或 3 列等距排布
- **限制**：最多 6 个模块/页

#### 7. 图文结合
- **适用**：解释复杂概念、展示结构
- **特征**：SVG 图形占 40-50%，配合文字说明
- **图形类型**：流程图、概念图、简化示意图

## 四、设计思维检查清单（多角色协同）

生成每一页 Slide 时，依次检查以下维度：

### Phase 1: 内容分析
- [ ] 核心主题是否明确？
- [ ] 信息类型已分类？（标题/列表/数据/金句/流程/对比）
- [ ] 逻辑关系已识别？（层级/因果/并列/时序）
- [ ] 内容密度是否需要拆页？

### Phase 2: 视觉层次
- [ ] 主次信息已区分？（字号/字重/颜色差异）
- [ ] 视线引导路径清晰？（F 型或 Z 型）
- [ ] 最重要信息一眼可见？
- [ ] 关键词/数字已做强调处理？

### Phase 3: 色彩与版式
- [ ] 色彩与主题调性匹配？
- [ ] 所有文本对比度达标？（≥ 4.5:1）
- [ ] 字体层级清晰？（标题/副标题/正文/说明）
- [ ] 行高和字间距舒适？

### Phase 4: 空间和谐
- [ ] 留白充足且有意图？
- [ ] 元素间距遵循 8px 网格？
- [ ] 页面视觉重量平衡？
- [ ] 无拥挤或空洞感？

### Phase 5: 跨页一致性
- [ ] 与前后页风格统一？（色彩/字体/元素风格）
- [ ] 装饰元素风格连贯？
- [ ] 亮度基调一致？（整体偏亮 or 偏暗）
- [ ] 品牌元素位置一致？

## 五、SVG 图形化数据表达

### 何时使用 SVG

| 场景 | 图表类型 | SVG 实现 |
|------|---------|---------|
| 占比/份额 | 饼图/环形图 | `<circle>` + `stroke-dasharray` |
| 趋势/变化 | 折线图 | `<polyline>` + `<circle>` 节点 |
| 对比量级 | 柱状图 | `<rect>` 组合 |
| 流程/步骤 | 流程图 | `<rect>` + `<line>` + `<text>` |
| 层级关系 | 金字塔 | `<polygon>` 梯形组合 |
| 关系网络 | 节点图 | `<circle>` + `<line>` |
| 进度/完成度 | 进度条 | `<rect>` overlay |

### SVG 设计原则

```
1. 颜色使用 CSS 变量：fill="var(--primary)"
2. 保持简洁：最多 5-6 个数据点/图表
3. 添加标签：<text> 标注关键数值
4. 动画适度：仅在入场时使用 stroke-dashoffset 动画
5. 响应式：viewBox 定义，不用固定宽高
```

### 简约饼图示例

```html
<svg viewBox="0 0 200 200" width="200" height="200">
  <!-- 背景圆 -->
  <circle cx="100" cy="100" r="80" fill="none"
    stroke="var(--bg-section)" stroke-width="24" />
  <!-- 数据弧 (72% = 0.72 × 502.65 ≈ 361.9) -->
  <circle cx="100" cy="100" r="80" fill="none"
    stroke="var(--primary)" stroke-width="24"
    stroke-dasharray="361.9 502.65"
    stroke-linecap="round"
    transform="rotate(-90 100 100)" />
  <!-- 中心数字 -->
  <text x="100" y="105" text-anchor="middle"
    font-size="28" font-weight="700"
    fill="var(--text-heading)">72%</text>
</svg>
```

### 简约柱状图示例

```html
<svg viewBox="0 0 400 200" width="400" height="200">
  <!-- Y 轴基线 -->
  <line x1="40" y1="180" x2="380" y2="180"
    stroke="var(--border)" stroke-width="1" />
  <!-- 柱形 -->
  <rect x="60" y="60" width="48" height="120" rx="4"
    fill="var(--primary)" opacity="0.3" />
  <rect x="130" y="40" width="48" height="140" rx="4"
    fill="var(--primary)" opacity="0.5" />
  <rect x="200" y="20" width="48" height="160" rx="4"
    fill="var(--primary)" opacity="0.7" />
  <rect x="270" y="10" width="48" height="170" rx="4"
    fill="var(--primary)" />
  <!-- 标签 -->
  <text x="84" y="195" text-anchor="middle"
    font-size="12" fill="var(--text-muted)">Q1</text>
  <text x="154" y="195" text-anchor="middle"
    font-size="12" fill="var(--text-muted)">Q2</text>
  <text x="224" y="195" text-anchor="middle"
    font-size="12" fill="var(--text-muted)">Q3</text>
  <text x="294" y="195" text-anchor="middle"
    font-size="12" fill="var(--text-muted)">Q4</text>
</svg>
```

## 六、主题语义-视觉映射扩展

在 color-system.md 的行业色彩惯例基础上，补充**语义层面**的映射：

### 情感意图 → 视觉属性

| 情感意图 | 色温 | 对比度 | 线条 | 留白 | 动效 |
|---------|------|-------|------|------|------|
| 激动人心 | 暖色 | 高 | 锐利/动感 | 中等 | Cinematic/Zoom |
| 严肃专业 | 冷色/中性 | 中 | 直线/几何 | 适度 | Fade/Slide |
| 温馨活泼 | 暖色 | 中低 | 柔和/圆润 | 充足 | Fade |
| 权威可靠 | 深色冷色 | 高 | 粗线/方正 | 适度 | Cut/Slide |
| 创新未来 | 冷色+霓虹 | 高 | 几何/断裂 | 中等 | Zoom/Flip |
| 宁静优雅 | 低饱和 | 低 | 细线/曲线 | 充足 | Fade |

### 品牌关键词 → 视觉倾向速查

| 关键词 | 色彩倾向 | 字体倾向 | 元素倾向 |
|-------|---------|---------|---------|
| 科技/智能 | 蓝紫渐变 | 细无衬线 | 几何线条、光效 |
| 速度/动感 | 红黑对比 | 粗无衬线 | 斜线、运动模糊 |
| 自然/环保 | 绿色系 | 圆润无衬线 | 有机曲线 |
| 奢华/品质 | 黑+金 | 衬线体 | 细线、金属质感 |
| 教育/知识 | 蓝+多色编码 | 清晰无衬线 | 图标、色块标签 |
| 儿童/童趣 | 高饱和多色 | 圆润粗体 | 圆角、阴影、Q版 |
| 极简/日式 | 白灰+木棕 | 超细无衬线 | 极细线条、留白 |

## 七、文字排版巧思

### 字号对比制造视觉韵律

```
大标题  ████████████   48-96px / 900 weight
副标题  ████████        24-32px / 600 weight
正文    ██████          16-18px / 400 weight
说明    ████            12-14px / 400 weight
```

**黄金比例建议**：相邻层级字号比 ≈ 1.5:1 至 2:1

### 关键词强调技巧

| 技巧 | CSS 实现 | 适用场景 |
|------|---------|---------|
| 颜色高亮 | `color: var(--primary)` | 关键术语 |
| 加粗 | `font-weight: 700` | 重要短语 |
| 背景色块 | `background: var(--primary); color: white; padding: 2px 8px;` | 标签/Tag |
| 下划线 | `border-bottom: 3px solid var(--accent)` | 定义/概念 |
| 大字号数字 | `font-size: 2.5em; font-weight: 800` | 统计数据 |
| 渐变文字 | `background: linear-gradient(...); -webkit-background-clip: text;` | Hero 标题 |

### 首字下沉（Editorial 风格专用）

```css
.editorial-content p:first-of-type::first-letter {
  float: left;
  font-size: 4em;
  line-height: 0.8;
  margin-right: 8px;
  font-weight: 700;
  color: var(--primary);
}
```

## 八、跨页一致性保障

### 全局视觉基调锁定

一旦封面页/首页确定了以下属性，后续所有页面必须保持一致：

| 属性 | 锁定规则 |
|------|---------|
| 背景亮度 | 亮色系 or 暗色系，全程统一 |
| 主色调 | 不变 |
| 强调色 | 不变 |
| 字体栈 | 不变 |
| 装饰元素风格 | 同一类型（几何/有机/线条） |
| Logo 位置 | 固定位置（通常左上或左下） |
| 导航点样式 | 统一大小、间距、active 样式 |

### 允许的页间变化

- 布局结构可以变化（根据内容类型）
- 配图内容不同但风格一致
- 信息密度可以不同但留白基调一致
- 装饰元素密度可微调但类型不变

## 九、配图与内容的关系规则

### 图片必须服务于叙事

| 原则 | 说明 |
|------|------|
| 相关性 | 图片必须与当页内容强相关 |
| 非装饰性 | 不为"好看"而配图，图片需承载信息 |
| 焦点明确 | 图片应有单一清晰的视觉焦点 |
| 构图互动 | 图片主体方向应引导向文字区域 |
| 情感一致 | 图片情绪与内容情感基调匹配 |

### 无图时的视觉补偿

当选择不使用配图时，通过以下方式保持视觉丰富度：
- SVG 图形化（图表、流程图、概念图）
- 色块分区（背景色彩变化）
- 文字排版韵律（字号对比、颜色变化）
- 几何装饰元素（线条、圆点、色带）
- 大数字/大引号作为视觉焦点
