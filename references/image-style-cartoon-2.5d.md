# 卡通 2.5D 风格 配图提示词指南

## 风格定义

友好、亲和、充满活力的插图风格，采用卡通化的 2.5D 等距视角。
适合：教育内容、职业指导、产品推广、用户引导等需要传达积极、乐观、易接近氛围的场景。
与严肃的商务科技风格不同，更注重情感连接和激励效果。

## 角色设计规范

### 基本原则
- **Q版比例**：头身比 1:2 或 1:1.5
- **圆润友好**：避免尖锐线条和角度
- **简化面部**：点状眼睛，简单嘴巴
- **表现力**：通过肢体动作传达情绪

### 角色类型

| 类型 | 姿态 | 服装 | 周边元素 |
|------|------|------|---------|
| 学习者 | 前倾好奇 | 浅色系 | 书本、问号 |
| 导师 | 自信引导 | 深色系 | 指示棒、徽章 |
| 成功者 | 举手庆祝 | 暖色系 | 奖杯、星星 |
| 协作者 | 面向同伴 | 互补色 | 连接线、对话泡 |

### 姿态情绪对照

| 情绪 | 姿态 | 周边元素 |
|------|------|---------|
| 好奇 | 前倾身体，手托下巴 | 问号气泡、放大镜 |
| 困惑 | 挠头，略后仰 | 问号、迷雾 |
| 自信 | 直立叉腰，微笑 | 徽章、证书 |
| 庆祝 | 双手举起，跳跃 | 奖杯、星星、彩带 |
| 协作 | 面向他人，手势交流 | 连接线、对话气泡 |

## 布局模式

### 模式 A - 进阶式（职业/学习路径）
```
Left (25%): Starting point - 新手角色 + 基础元素
Center (50%): Journey - 阶梯/道路 + 里程碑
Right (25%): Goal - 成功角色 + 成就元素
Visual flow: 从左到右上升趋势
```

### 模式 B - 中心辐射式（产品功能/特点）
```
Center (40%): Hero element - 产品/吉祥物
Surrounding (60%): Feature nodes - 功能图标环绕
Layout: 圆形或星形排列
```

### 模式 C - 场景叙事式（工作流程）
```
Multiple platforms at different heights
Characters interacting across platforms
Objects/data flowing between scenes
```

### 模式 D - 对比展示式（前后对比）
```
Left (40%): Before - 灰暗色调 + 困难场景
Center (20%): Transformation - 箭头/转换元素
Right (40%): After - 明亮色调 + 成功场景
```

## 元素库

### 平台元素
- Rounded rectangular platforms（圆角矩形平台）
- Circular stages（圆形舞台）
- Stepped platforms（阶梯平台）
- Floating islands（漂浮岛屿）

### 道具
- **教育类**：Books, laptop, pencils, lightbulb, certificate
- **工作类**：Desk, computer, coffee cup, briefcase
- **成就类**：Trophy, medal, star badge, crown
- **科技类**：Smartphone, cloud icon, gear, rocket, robot

### 装饰元素
- Stars (sparkles), Hearts, Confetti
- Speech/Thought bubbles
- Arrows, Checkmarks
- Dots/circles

## 与红黑白科技风对比

| 维度 | 红黑白科技 | 卡通 2.5D |
|------|-----------|----------|
| 情绪 | 专业、理性 | 友好、乐观 |
| 配色 | 红+黑+白 | 多彩和谐 |
| 角色 | 黑色剪影 | Q版卡通 |
| 形状 | 锐利几何 | 圆润柔和 |
| 阴影 | 硬阴影 | 软阴影 |
| 装饰 | 电路线、几何 | 星星、爱心 |
| 适用 | 技术/B2B | 教育/B2C |

**选择建议**：
- 传达专业性、技术性 → 红黑白科技
- 传达友好性、激励性 → 卡通 2.5D
- 目标受众是技术人员 → 红黑白
- 目标受众是普通用户 → 卡通 2.5D
